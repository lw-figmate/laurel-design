### General Component Usage

#### Common Props

Most Laurel Design components accept these common props:
- `className`: String for additional CSS classes (use sparingly — prefer built-in variants)
- `disabled`: Boolean to disable interactive components
- `data-*`: Custom data attributes are always allowed
- Components extend the types of their base HTML element (e.g., Button accepts all `<button>` props)

#### Sizing

Most components support a `size` prop:
- `"sm"` — Small
- `"md"` — Medium (default)
- `"lg"` — Large

#### Component Selection Guide

- **Select vs Input vs Combobox**: Use `Select` for 4–20 predefined options. Use `Input` for freeform text values. Use `Combobox` for searchable/filterable options or >20 items.
- **Dialog vs Drawer vs Sheet**: Use `Dialog` for important actions requiring user attention. Use `Drawer` for side panel content or navigation. Use `Sheet` for bottom sheets on mobile or secondary panels.
- **Popover vs Tooltip vs HoverCard**: Use `Tooltip` for simple hover hints. Use `Popover` for interactive content triggered by click. Use `HoverCard` for rich preview content on hover.
- **Alert vs Banner vs Toast**: Use `Alert` for inline status messages. Use `Banner` for full-width announcements. Use `Toast` for brief, auto-dismissing notifications.

### Hooks

The following hooks are available:
- `useTheme()` — Access and control the current theme
- `useMediaQuery(query)` — Subscribe to CSS media query matches
- `useDisclosure()` — Toggle open/close state for modals, drawers, etc.
- `useClickOutside(ref, handler)` — Detect clicks outside a ref element
- `useFocusTrap(ref)` — Trap keyboard focus within a container
- `useFormField()` — Access form field context (label, error, required state)

### Utility Functions

- `cn(...classes)` — Merge class names, filtering falsy values
- `composeRefs(...refs)` — Compose multiple refs into one
- `validators` / `validate` / `validateForm` — Form validation utilities
